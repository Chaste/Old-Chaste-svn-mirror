Chaste File Formats

.axi
.dat
Fixed-width space-separated column data file format, written by the ColumnDataWriter class, and compatible with Gnuplot.
  • First line is a header, of the form Name1(units1) Name2(units2) ... NameN(unitsN).
  • Subsequent lines give the data. Columns are padded with spaces to a fixed width, determined by the precision argument to the writer constructor.
.edge
Edge file for Triangle or Tetgen format meshes.
  • First line: <# of edges> <# of boundary markers (0 or 1)>
  • Following lines: <edge #> <endpoint node> <endpoint node> [boundary marker]
  • Lines starting with a hash (#) are comments
.ele
Element file for Triangle or Tetgen format meshes.
  • First line: <# of elements> <nodes per element> <# of attributes>
  • Remaining lines: <element #> <node> <node> <node> ... [attributes]
  • Lines starting with a hash (#) are comments
.epi
Used to specify a list of node numbers (one column) of the nodes belonging to the epicardial surface.
.endo
Used to specify a list of node numbers (one column) of the nodes belonging to the endocardial surface.
.exnode
.exelem
CMGUI node and element formats respectively, see the Cmgui guide.
.face
Faces file for Tetgen format meshes.
  • First line: <# of faces> <boundary marker (0 or 1)>
  • Remaining lines: <face #> <node> <node> <node> [boundary marker]
  • Lines starting with a hash (#) are comments

.node
Nodes file for Triangle or Tetgen format meshes.
  • First line: <# of vertices> <space dimension> <# of attributes> <# of boundary markers (0 or 1)>
  • Remaining lines: <vertex #> <x> [<y>] [<z>] [attributes] [boundary marker]
  • Lines starting with a hash (#) are comments
